# UK Road Collision Hotspot Map

An interactive map of every reported road collision in Great Britain (2020–2025), built using free open government data from the Department for Transport (DfT).

---

## What the App Does

- Plots **551,881 road collisions** on an interactive UK map
- Filter by severity: **Fatal** (markers) · **Serious** (markers) · **Slight** (heatmap) · **All** (heatmap)
- **Postcode search** — type any UK postcode, pick a radius (0.5 / 1 / 3 / 5 miles), get a breakdown of collisions nearby
- **Clear button** — resets map to default state with full summary stats
- Header shows live collision counts per severity type and year range
- Dark themed, mobile-friendly

---

## Data Source

**DfT STATS19 Road Safety Open Data**
Published by the Department for Transport. Every reported road collision attended by police in England, Scotland, and Wales.

| File | Coverage |
|---|---|
| `dft-road-casualty-statistics-collision-last-5-years.csv` | 2020–2024 (finalised) |
| `dft-road-casualty-statistics-collision-provisional-2025.csv` | 2025 (provisional, updated periodically) |

- **Free** — Open Government Licence, no API key needed
- **Not included**: Northern Ireland (separate system), unattended collisions
- **Severity**: 1 = Fatal, 2 = Serious, 3 = Slight

## Architecture

```
GitHub repo (no accidents.json)
        ↓ git push
Render build server:
  pip install -r requirements.txt
  python process_data.py
    → downloads CSVs from DfT (~100MB)
    → generates accidents.json (~20MB)
        ↓
CDN serves: index.html + accidents.json
        ↓
Browser runs everything:
  Leaflet.js map
  MarkerCluster for Fatal/Serious
  Heatmap for Slight/All
  Postcode lookup via postcodes.io (free, no key)
  Web Worker for postcode filtering (off main thread)
```

No backend. No database. No server. Just files on a CDN.

## How to Redeploy

### Code change (e.g. update the map UI)

```bash
# Edit index.html or any file
git add .
git commit -m "your change description"
git push
```

Render detects the push and auto-deploys. Takes ~3-4 minutes (re-downloads STATS19 data). No action needed on Render.

### Data refresh (new STATS19 data released)

DfT releases updated data annually (~June/July). To pick up new data:

```bash
# Just push any commit — Render re-downloads fresh data on every build
git commit --allow-empty -m "trigger data refresh"
git push
```

Or manually trigger a deploy from the Render dashboard → **Manual Deploy**.

### Run locally

```bash
cd uk-accident-map
pip3 install -r requirements.txt
python3 process_data.py          # downloads CSVs + generates accidents.json
python3 -m http.server 8080
# open http://localhost:8080
```

### Re-run data processing with a local CSV

```bash
python3 process_data.py path/to/your-file.csv
```

## Challenges Encountered During Deployment

|---|---|---|---|
